# 微博正文采集

## 功能目的

按微博正文链接批量采集单条公开博文详情。

## 输入与运行选项

- 输入格式：`https://weibo.com/数字用户ID/博文ID`。
- 支持逐条输入与批量粘贴；每个链接是独立任务。
- 支持默认 `1000 - 6000 ms` 随机链接间隔。
- 不提供循环监控；每个链接每次只读取一条最新详情。
- 不进行微博登录状态前置检查。

## 执行流程

1. 打开正文链接。
2. 等待正文、媒体和互动统计连续稳定。
3. 读取公开详情并保存；同一正文再次采集时覆盖本地旧数据。

## 数据字段

`postId`、`postUrl`、`visibility`、`author`、`authorUrl`、`authorAvatar`、`text`、`publishedAt`、`source`、`reposts`、`comments`、`likes`、`topics`、`mentions`、`contentLinks`、`mediaUrls`、`capturedAt`。

## 数据与记录

- 支持按链接和状态筛选运行记录，并在任务明细中显示错误原因。
- 发布时间只从正文卡片的日期链接读取；两位年份会补全为四位年份，例如 `26-7-16 11:16` 转为 `2026-7-16 11:16`。
- 数据最多保留 3000 条；每种状态最多保留 200 条记录。
- 支持数据筛选、复制筛选后的 JSON 与导出 CSV。

## 注意事项

仅读取页面公开内容，不执行转发、点赞、评论等互动操作。

## 统一传输字段

每条新正文还会写入 `canonical`：`entityType` 为 `content`，`contentType` 为 `post`。标准字段统一保存作者、正文、发布时间、话题、提及、媒体与互动指标；微博可见范围、来源、关联链接、功能原始字段及完整页面文本保留在 `platformExtra`。字段中文说明与示例见[统一传输数据模型](../data-schema.md)。
