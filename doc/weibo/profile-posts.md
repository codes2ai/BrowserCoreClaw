# 微博博主博文采集

## 功能目的

按微博博主公开主页采集博文卡片，并保留主页中的原始展示顺序。

## 输入与运行选项

- 登录前置：进入功能时自动检测当前 Chrome Profile 的微博登录状态；未登录时不会显示运行页面。
- 输入格式：`https://weibo.com/u/数字ID`。
- 支持多主页逐条输入或批量粘贴；每个主页是独立任务。
- 每主页结果数：1–100 条，默认 10 条；已保存的旧默认值会自动迁移一次。
- 执行间隔：默认 `1000 - 6000 ms` 随机区间。
- 支持循环监控。

## 执行流程

1. 使用独立临时页检测当前 Chrome Profile 的微博登录状态，检测完成后自动关闭临时页。
2. 未登录时打开专用微博登录页，完成登录后回到扩展重新检测。
3. 登录确认成功后进入功能，任务使用独立页面打开目标主页。
4. 等待资料区和博文列表连续稳定。
5. 读取首屏博文；不足设定数量时按视口高度的约 55%–90% 进行短距离平滑滚动。
6. 每次滚动后随机停顿 2.2–4.5 秒，等待页面内容加载稳定后再继续。
7. 到达页面底部或连续多轮没有新博文时停止滚动，避免密集请求与无限加载。

## 数据字段

`postId`、`author`、`text`、`publishedAt`、`source`、`reposts`、`comments`、`likes`、`mediaUrls`、`url`、`capturedAt`。

## 数据与记录

- 每个主页、每轮均生成单独记录，可按主页和状态筛选。
- 数据按主页与博文 ID/链接去重，最多保留 3000 条。
- 每种状态最多保留 200 条记录，支持任务明细、数据筛选、复制筛选后的 JSON 和导出 CSV。

## 注意事项

登录状态是运行本功能的必要条件。登录过期或页面出现安全验证时，本主页会记录失败原因；完成登录或验证后重新进入功能检测即可。

## 统一传输字段

每条新博文还会写入 `canonical`：`entityType` 为 `content`，`contentType` 为 `post`。标准字段包括作者、正文、发布时间、媒体、话题、互动指标及来源主页；微博专有来源、可见范围和关联链接保留在 `platformExtra`，`rawPageText` 保存完整可见页面原文。字段中文说明与示例见[统一传输数据模型](../data-schema.md)。
