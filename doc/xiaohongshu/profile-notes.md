# 小红书博主博文采集

## 功能目的

按小红书博主公开主页采集笔记卡片，不采集博主资料。

## 登录要求

运行前需要通过小红书登录检测。登录失效或出现安全验证时，当前主页会失败并保留错误信息。

## 输入与运行选项

- 输入格式：小红书 `/user/profile/<id>` 主页链接。
- 支持多个主页、批量编辑、每主页 1–100 条笔记。
- 支持默认 `1000 - 6000 ms` 随机主页间隔和循环监控。

## 执行流程

1. 打开目标主页并等待笔记列表稳定。
2. 按页面原始顺序读取笔记卡片。
3. 初始列表不足时滚动补齐；连续多次没有新笔记时停止。

## 数据字段

`profileUrl`、`pageOrder`、`noteId`、`title`、`author`、`likes`、`cover`、`url`、`capturedAt`。

## 数据与记录

- 每个主页独立记录，可按主页和状态筛选。
- 数据以笔记链接去重，最多 3000 条。
- 每种状态最多 200 条记录，支持任务明细、数据筛选、复制筛选后的 JSON 和导出 CSV。

## 统一传输字段

每条新主页笔记还会写入 `canonical`：`entityType` 为 `content`，`contentType` 为 `note`。统一字段保存笔记 ID、标题、作者、封面、点赞、来源主页和页面顺序；页面原始字段与完整可见页面文本保留在 `platformExtra`。字段中文说明与示例见[统一传输数据模型](../data-schema.md)。
