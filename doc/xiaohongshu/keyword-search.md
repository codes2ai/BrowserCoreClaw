# 小红书关键词搜索

## 功能目的

按关键词搜索小红书笔记，应用页面可见筛选条件后采集结果卡片，并保持页面展示顺序。

## 登录要求

首次运行会检查当前 Chrome Profile 的小红书登录状态。未登录时会展示登录门禁；请在小红书标签页自行完成登录、扫码或验证码，再返回扩展点击“重新检测”。扩展不会保存或代填账号凭据。

## 输入与运行选项

- 关键词：支持逐条编辑、批量编辑和 JSON 编辑。
- 每关键词笔记数：1–100 条。
- 筛选：排序、笔记类型、发布时间、搜索范围、位置距离（以当前页面支持项为准）。
- 关键词随机间隔：默认 `1000 - 6000 ms`。
- 支持循环监控。

## 执行流程

1. 确认登录状态后，逐条打开关键词搜索页。
2. 应用选定筛选条件并等待页面筛选完成。
3. 读取首屏笔记；不足时滚动加载，直至达到数量或不再出现新卡片。

## 数据字段

`keyword`、`pageOrder`、`cover`、`title`、`description`、`author`、`publishedAt`、`likes`、`url`、`capturedAt`。

## 数据与记录

- 每个关键词独立记录；可按关键词与状态筛选。
- 数据最多保存 3000 条，每种状态最多保存 200 条记录。
- 支持任务明细、数据筛选、复制筛选后的 JSON 与导出 CSV。

## 统一传输字段

每条新笔记卡片还会写入 `canonical`：`entityType` 为 `content`，`contentType` 为 `note`。统一字段保存关键词、页面顺序、标题、摘要、作者、封面、发布时间和点赞；小红书筛选页的原始字段及完整可见页面文本保存于 `platformExtra`。字段中文说明与示例见[统一传输数据模型](../data-schema.md)。
