# 小红书正文采集

## 功能目的

按小红书笔记正文链接批量采集单条笔记详情。

## 登录要求

进入功能前需要通过小红书登录检测。登录失效或出现安全验证时，任务会记录具体错误，不会继续写入不完整数据。

## 输入与运行选项

- 支持 `/search_result/<note-id>`、`/explore/<note-id>`、`/discovery/item/<note-id>` 和博主主页中的笔记链接。
- 建议保留原链接中的 `xsec_token`、`xsec_source` 等查询参数。
- 支持逐条输入、批量粘贴、分页展示和并发任务；默认并发数为 1。
- 支持默认 `1000 - 6000 ms` 随机链接间隔。
- 不提供循环监控；每个链接每次只读取一条详情。

## 执行流程

1. 通过小红书登录状态检测。
2. 在独立后台标签页打开正文链接，并核对页面笔记 ID 与输入链接一致。
3. 连续确认作者、正文、媒体和互动数据稳定。
4. 读取详情并保存；链接不存在、已删除或跳转到其他页面时作为失败记录，不写入数据。

## 数据字段

`noteId`、`noteType`、`title`、`description`、`author`、`authorId`、`authorUrl`、`authorAvatar`、`publishedAt`、`ipLocation`、`likes`、`favorites`、`comments`、`shares`、`topics`、`cover`、`imageUrls`、`videoUrls`、`postUrl`、`capturedAt`。

## 数据与记录

- 同一笔记再次采集时以 `noteId` 去重并更新最新详情。
- 支持按正文链接和状态筛选运行记录，任务明细显示耗时、结果数量、新增数量和错误原因。
- 数据页支持字段筛选、表头设置、筛选后 JSON 预览与一键复制，以及筛选后 CSV 导出。
- 存储条数遵循设置页的全局 Limit 配置。

## 注意事项

仅读取当前页面公开可见内容，不执行点赞、收藏、评论、分享或关注操作。平台页面结构变化后，可能需要同步更新 `page-extract.js` 中的详情选择器。
